# Visión General: Las 8 Leyes de Evolución como Sistema

## Introducción

Genrich Altshuller, tras analizar decenas de miles de patentes, descubrió que los sistemas
técnicos no evolucionan al azar. Siguen **8 leyes objetivas** que describen las condiciones
de viabilidad, la dirección y las tendencias de su evolución. Estas leyes permiten
**diagnosticar** en qué etapa se encuentra un sistema y **predecir** sus próximos pasos
evolutivos.

## Clasificación en tres grupos

Altshuller organizó las 8 leyes en tres categorías fundamentales:

```
┌─────────────────────────────────────────────────────────────┐
│              LEYES DE EVOLUCIÓN DE SISTEMAS TÉCNICOS        │
├───────────────┬───────────────────┬─────────────────────────┤
│   ESTÁTICAS   │   CINEMÁTICAS     │       DINÁMICAS         │
│  (existencia) │   (dirección)     │      (tendencias)       │
├───────────────┼───────────────────┼─────────────────────────┤
│ 1. Completitud│ 4. Idealidad      │ 7. Transición a         │
│ 2. Conductivi-│ 5. Desarrollo     │    micronivel           │
│    dad energ. │    desigual       │ 8. Aumento de           │
│ 3. Coordinac. │ 6. Transición a   │    dinamismo y          │
│    de ritmos  │    supersistema   │    controlabilidad      │
└───────────────┴───────────────────┴─────────────────────────┘
```

### Leyes estáticas: condiciones de viabilidad

Definen los **requisitos mínimos** para que un sistema técnico sea viable. Sin cumplir
estas condiciones, el sistema simplemente no funciona o no sobrevive:

| Ley | Nombre | Pregunta clave |
|-----|--------|----------------|
| 1 | [Completitud de las partes](01-ley-completitud.md) | ¿Tiene el sistema sus 4 partes esenciales? |
| 2 | [Conductividad energética](02-ley-conductividad.md) | ¿Fluye la energía libremente por todo el sistema? |
| 3 | [Coordinación de ritmos](03-ley-coordinacion.md) | ¿Están sincronizados los ritmos de todas las partes? |

### Leyes cinemáticas: dirección de evolución

Describen **hacia dónde** evolucionan los sistemas, sin especificar cómo:

| Ley | Nombre | Pregunta clave |
|-----|--------|----------------|
| 4 | [Aumento de la idealidad](04-ley-idealidad.md) | ¿Se acerca el sistema al resultado ideal? |
| 5 | [Desarrollo desigual](05-ley-desarrollo-desigual.md) | ¿Qué subsistema limita la evolución del conjunto? |
| 6 | [Transición a supersistema](06-ley-transicion-supersistema.md) | ¿Puede el sistema integrarse en un sistema mayor? |

### Leyes dinámicas: tendencias de evolución

Describen las **tendencias concretas** que siguen los sistemas al evolucionar:

| Ley | Nombre | Pregunta clave |
|-----|--------|----------------|
| 7 | [Transición al micronivel](07-ley-transicion-micronivel.md) | ¿Puede el elemento de trabajo operar a nivel micro? |
| 8 | [Dinamización y controlabilidad](08-ley-dinamizacion.md) | ¿Puede el sistema volverse más flexible y adaptativo? |

## Tabla resumen de las 8 leyes

| # | Ley | Grupo | Esencia | Ejemplo |
|---|-----|-------|---------|---------|
| 1 | Completitud | Estática | 4 partes mínimas: motor, transmisión, herramienta, control | Un martillo necesita mano (motor), brazo (transmisión), cabeza (herramienta), ojo (control) |
| 2 | Conductividad | Estática | La energía debe fluir sin bloqueos | Circuito eléctrico: corte en cable = sistema no funciona |
| 3 | Coordinación | Estática | Ritmos y frecuencias deben estar sincronizados | Caja de cambios adapta RPM del motor a velocidad de ruedas |
| 4 | Idealidad | Cinemática | Evolución hacia más función con menos recursos | Teléfono fijo → móvil → smartphone (más funciones, mismo objeto) |
| 5 | Desarrollo desigual | Cinemática | Subsistemas evolucionan a ritmos diferentes | Batería limita al smartphone más que el procesador |
| 6 | Supersistema | Cinemática | Integración en sistemas mayores | Teléfono + cámara + GPS → smartphone |
| 7 | Micronivel | Dinámica | De mecanismos macro a micro | Corte: hoja → láser → chorro de agua → plasma |
| 8 | Dinamización | Dinámica | De rígido a flexible a inteligente | Ala fija → flaps → ala deformable |

## Uso práctico de las leyes

Las leyes de evolución se aplican en tres modos principales:

1. **Diagnóstico**: Determinar en qué etapa evolutiva se encuentra un sistema actual.
   Esto complementa el análisis realizado en [ARIZ-85C](../07-ariz/09-parte-9-proceso.md)
   durante la formulación del problema.

2. **Predicción**: Anticipar los próximos pasos evolutivos del sistema. Cada ley indica
   una dirección concreta de mejora. Esto se conecta con los
   [40 Principios Inventivos](../02-principios-inventivos/00-indice-principios.md) como
   mecanismos para implementar la evolución predicha.

3. **Generación de conceptos**: Usar las leyes como guía para generar soluciones
   innovadoras durante el [análisis Su-Campo](../05-analisis-su-campo/02-modelos-basicos.md)
   y la aplicación de [soluciones estándar](../06-soluciones-estandar/01-clase-1-construir.md).

## Relación con otras herramientas TRIZ

```
Leyes de Evolución
       │
       ├── Diagnóstico ──→ Análisis Su-Campo (modelar el sistema actual)
       ├── Predicción ───→ Matriz de Contradicciones (resolver bloqueos)
       └── Concepto ─────→ 40 Principios + Soluciones Estándar (implementar)
```

Las leyes de evolución representan el nivel más alto de abstracción en TRIZ. Proporcionan
el marco estratégico dentro del cual operan todas las demás herramientas: los
[parámetros de ingeniería](../03-parametros-ingenieria/00-indice-parametros.md), la
[matriz de contradicciones](../04-matriz-contradicciones/00-como-usar-la-matriz.md) y el
proceso completo de [ARIZ](../07-ariz/00-que-es-ariz.md).
